# Library Directory Guide

This directory is reserved for business-facing document assets, traditional forms, commodity-specific operational files, and public content-service source materials.

## Use this directory for

- market-specific form libraries
- downloadable document assets
- shared operational templates
- commodity rollout folders
- future content-service publication assets

## Do not use this directory for architecture source-of-truth

Architecture and design baseline documents must live under:

- [docs/design](/C:/Users/mos/Documents/GitHub/aegis-broker/docs/design)

This split keeps implementation design and business document assets on one predictable path each:

- `docs/design`: architecture, plans, and preserved design decisions
- `library/`: commodity files, forms, templates, and future content-service assets
