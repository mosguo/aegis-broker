內容服務架構

以經紀商網站的服務、商品、團隊與聯絡資訊為前台內容骨架，並接上本專案的公開文件與傳統業務表單服務。

Brokerage Content

  • 經紀服務、流程顧問、市場資訊與商品目錄採公開唯讀展示。
  • 內容主軸對應 About、Services、Commodities、Contact 四類頁面。
  • 訪客以預設 workspace 讀取公開內容，不要求先登入。

Document Services

  • 商品交易文件庫依市場別管理：咖啡豆、原油、金屬、共用制度文件。
  • 公開文件提供瀏覽、下載、版本說明；受控文件保留事件與稽核軌跡。
  • 後續可擴充表單申請、文件發佈、下載授權與流程追蹤。

Zeabur reference

  • Bottom-left user operations live inside an account card and overflow menu.
  • Primary labels are about 14px and secondary role text is about 12px.
  • Status signals stay visible without depending on manual action.

Current runtime

  • The bottom-left card now exposes profile, workspace, session, and login actions.
  • Name uses 14px and role uses 12px to match Zeabur's footer hierarchy.
  • Three system status groups now query on load and keep polling automatically.

傳統業務表單服務

整合目前 `library/` 資料夾的商品交易文件與傳統表單，作為內容服務的文件中心與後續數位化流程入口。

Market Libraries

  • 咖啡豆市場：詢價單、報價單、採購單、樣品追蹤、庫存與物流文件。
  • 原油市場：沿用同一組核心、擴充、進階文件分類，便於跨品類複用。
  • 金屬市場：同樣具備核心表單、作業文件與稽核/管理模板。

Shared Governance

  • 共用主檔與制度文件提供供應商、原因碼、商品規格、版本規則與狀態流轉規則。
  • 內容服務需先做文件索引、分類、版本與可見性管理，再逐步接入流程化送審。
  • 所有受控發佈與下載申請，後續都應寫入 event_store 與 audit_chain。

Connection Settings

Status cards auto-poll every 30 seconds and can also be refreshed manually.
